1) Project Summary

UniSocial (MFP) is a multi-tenant social media management platform for organizations to connect accounts, schedule content, and track performance in one workspace.

  • Auth + onboarding + org roles
  • Multi-platform social account integrations
  • Scheduling + async publishing
  • Analytics dashboards
  • News feed + AI content assistance
  • Audit + platform admin controls

2) Core Product Features

  • Email OTP authentication and secure session lifecycle
  • Organization management with role-based access
  • Social account OAuth and publishing target sync
  • Post scheduling and cross-platform publishing
  • Analytics overview and platform-specific insights
  • Industry-driven news feed
  • AI-assisted post generation
  • Platform Admin: users, organizations, posts, social accounts, industries, news sources, audit logs

3) Technology Stack

LayerPrimary Tools
FrontendReact 19, Vite, Tailwind, Axios, Recharts
BackendDjango, DRF, SimpleJWT
DataPostgreSQL, Redis
AsyncCelery, Celery Beat, Flower
InfraDocker Compose, Nginx, Certbot
StorageAWS S3 (django-storages)

4) Architecture (High Level)

  1. Frontend SPA calls backend APIs.
  2. Nginx handles HTTPS and proxies to Django.
  3. Django executes auth/business logic and DB access.
  4. Celery workers process background jobs.
  5. Redis supports caching and queue transport.

5) Security Essentials

  • JWT access token + HttpOnly refresh cookie
  • Refresh rotation and blacklist
  • OTP verification with cooldown and attempt controls
  • Encrypted social provider token storage
  • CORS/CSRF and secure cookie controls
  • Organization-scoped authorization + role checks
  • Immutable audit logs

6) Session Essentials

  • Access token in Bearer header from frontend API client
  • Refresh token in cookie, refreshed via /api/v1/auth/token/refresh/
  • Frontend refresh de-duplication prevents race conditions
  • 401 refresh failure clears session and redirects to login

7) Scheduling and Publishing Essentials

  • Scheduler uses publishing_target_ids to map each post to exact account/provider
  • Backend validates target ownership and schedule time
  • Celery dispatches due posts every minute
  • Provider publishers handle Instagram/LinkedIn/YouTube publish logic
  • Status lifecycle: pending -> processing -> success/failed

8) Analytics Essentials

  • Periodic analytics sync jobs for published posts
  • Current metrics + historical snapshots
  • Redis-cached dashboard and platform analytics endpoints

9) API Groups

  • /api/v1/auth/
  • /api/v1/organizations/
  • /api/v1/social/
  • /api/v1/posts/
  • /api/v1/analytics/
  • /api/v1/news/

  • /api/v1/ai/
  • /api/v1/admin-panel/

10) Deployment and Operations Essentials

  • Deploy via Docker Compose
  • Nginx terminates TLS and proxies to web service
  • Health endpoints: /health/, /ready/
  • API docs: /swagger/, /redoc/
  • Queue monitor: Flower

11) Current Focus Areas

  • Ongoing mobile responsiveness refinements
  • LinkedIn analytics limitations due provider permissions
  • CI/CD and test coverage expansion
